A* (A-Star) is a heuristic-guided pathfinding algorithm. It is highly efficient because it estimates the distance to the target using a heuristic function (Manhattan Distance on grid), minimizing the total cost: f(n) = g(n) + h(n).

Welcome to Algoverse! 🌌

This sandbox lets you explore and visualize classic graph algorithms running in real-time. Here is how to use it:

  • Drag & Drop: Drag the green star ★ to change the starting location, and the red star ★ to set the destination.
  • Draw Terrain: Click and drag on empty cells to paint **Walls** (impassable barriers) or **Weights** (slows movement down, resembling swamps or mud).
  • Generate Mazes: Use the Maze & Terrain dropdown to generate complex labyrinth systems like Recursive Division or randomized DFS.
  • Run and Compare: Pick an algorithm (A*, Dijkstra, BFS, DFS) and click **Visualize Algorithm** to see it seek the shortest path!
Tip: Holding the W key on your keyboard while dragging automatically switches the brush to **Weight mode**. Pressing C clears the path!

2. System Architecture

01

Zero-Dependency Core

Developed using raw HTML5 grids and vanilla ES6+ Javascript. Zero framework overhead yields rendering speeds up to 60fps.

02

Stateful Node Grid Model

Each cell represents a dynamic graph node storing wall properties, heuristic costs, accumulated path weights, and search history states.

03

Hardware-Accelerated UI

Uses modern CSS3 grid and keyframe animations combined with hardware translation handles for lag-free node ripples and glowing paths.

3. Heuristic Solver: A* Search

f(n) = g(n) + h(n)

g(n): Traversal cost from start to current node.

h(n): Heuristic estimation to target (using Manhattan Distance).

  • Optimal & Admissible: Since Manhattan distance never overestimates the actual distance, A* is guaranteed to find the shortest path.
  • Min-Heap Implementation: Optimized using a custom Binary Min-Heap Priority Queue to extract minimum nodes in O(log V).
  • Complexities:
    • Time Complexity: O((V + E) log V)
    • Space Complexity: O(V)

4. Weighted Solver: Dijkstra's Algorithm

h(n) = 0

Dijkstra's acts as an uninformed search. It explores in uniform circular waves, comparing cost purely based on distance from start node.

  • Weighted Pathfinder: Excellent for routing through terrains. Highlights how paths curve to bypass weighted nodes.
  • Swamp Mechanics: Highlights the concept of edge weights. In the sandbox, standard nodes cost 1, whereas weight mud nodes cost 5.
  • Complexities:
    • Time Complexity: O((V + E) log V)
    • Space Complexity: O(V)

5. Unweighted Traversals: BFS & DFS

Characteristic Breadth-First Search (BFS) Depth-First Search (DFS)
Data Structure FIFO Queue LIFO Stack / Recursion
Exploration Style Concentric ripples, exploring layer-by-layer Winding trails, exploring deep branches first
Path Optimality Optimal for unweighted steps (ignores weights) Non-optimal (can construct extremely long paths)
Time Complexity O(V + E) O(V + E)

6. Maze & Terrain Engineering

Recursive Division

Divide-and-Conquer: Recursively splits chambers horizontally or vertically, drawing solid wall divisions while keeping one cell open as a passage.

Randomized DFS Carver

Labyrinth Backtracker: Fills the grid with walls, then backtracks using a stack to break down walls two steps away. Creates winding corridors with exactly one solution path.

7. Summary & Core Highlights

Instant Path Recalculation

Recalculates solution paths instantly (in 0ms) when start/target nodes are dragged after an initial search, illustrating real-time path warps.

Custom Data Structures

Avoids slow JavaScript array sort operations. Employs a custom Binary Min-Heap Priority Queue for optimal pathfinding performance.

Responsive Interface

Modular glassmorphic CSS rules adapt to screen sizes from widescreen laptops to compact mobile viewports.
